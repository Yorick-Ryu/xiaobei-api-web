# Product

## Register

brand

## Users

主要面向两类用户，以第一类为主：

1. **初入 AI 科研的研究生**：刚开始尝试把 AI 工具用到论文、代码、文献处理中，对工具链不熟悉，需要引导和信任建立。
2. **已有 Claude / ChatGPT 使用经验的科研者**：知道 Claude Code 和 ChatGPT Codex 的能力，在寻找稳定可靠的国内接入方式。

使用场景：在宿舍或实验室，用 Claude Code、ChatGPT Codex 辅助写论文、调试代码、整理文献，随时需要稳定高速的 API 连接。

## Product Purpose

小北科研API 是一个面向研究生、博士生和科研工作者的 Claude 与 ChatGPT API 中转站。提供满血（不降智）、稳定高速的 API 接入，让用户不需要折腾网络和账号，专注于科研任务本身。

背后是全网 10 万+ 粉丝、长期输出 AI 科研内容的创作者，以垂直信任背书服务。

成功标准：用户理解服务、信任创作者，并点击接入或查看套餐。

## Brand Personality

**专业、高效、有温度**

像一个懂科研流程的老师兄/老师姐，知道你在卡什么、需要什么，直接给方案，不卖弄技术，也不用力过猛。

情感目标：让用户感到「这个服务理解我、靠得住」，而不是「又一个中间商」。

## Anti-references

- **通用 SaaS 落地页模板**：大标题 + 三列特性卡片 + 绿色按钮，千页一律。
- **暗黑霓虹科技风**：黑底 + 发光色 + 粒子动画，像加密货币项目或游戏外设。
- **学术 / 政府官网**：过于严肃保守，缺乏设计感，让人联想到 PPT 模板。
- **过度营销风格**：堆满徽章、倒计时、「限时优惠」，显得廉价不可信。
- **紫色渐变 + emoji 装饰**：AI 产品的陈词滥调，毫无辨识度。

## Design Principles

1. **信任先于转化**：页面的第一任务是让用户相信这个服务，点击是信任之后的自然结果，而不是被催促的行为。
2. **克制即专业**：适合科研用户的界面不需要花哨，用留白、节奏和精准的文案表达专业感。
3. **场景而非功能**：用「写论文、跑代码、读文献」等真实任务描述服务，而不是罗列 API 参数和技术指标。
4. **创作者人格可见**：10万+粉丝和长期内容积累是真实的信任来源，设计应当让这个背书自然呈现，而不是埋在角落。
5. **中文优先**：页面受众是中文科研圈，设计和排版节奏应当贴合中文阅读习惯。

## Accessibility & Inclusion

- 遵循 WCAG 2.1 AA 标准，文字对比度达标。
- 支持 `prefers-reduced-motion`，动画降级已在代码中实现。
- 无特殊已知用户需求，键盘可访问性和语义化 HTML 基本保障即可。
